# 数据库建表数据结构 v3

> **项目名称**：基于 RBAC 的高校学生信息动态脱敏系统
> **技术栈**：MySQL 8.0 + Spring Boot 3 + Vue 3
> **架构模式**：混合架构（Java 鉴权编排 + MySQL 脱敏计算 + 触发器审计）

## 一、表设计总览

整个系统共 **14 张表**，加 **6 个数据库对象**，按照职责划分为五大体系：

| 体系 | 表数量 | 表名 |
|---|---|---|
| 权限体系 | 5 | `sys_user`, `sys_role`, `sys_permission`, `sys_user_role`, `sys_role_permission` |
| 业务体系 | 2 | `student_info`, `student_score` |
| 脱敏配置体系 | 4 | `sensitive_field`, `masking_type_dict`, `masking_policy`, `masking_rule_assignment` |
| 审计体系 | 3 | `access_log`, `rule_change_log`, `abnormal_access` |

| 对象类型 | 数量 | 名称 |
|---|---|---|
| 函数 | 1 | `FN_APPLY_MASK` |
| 存储过程 | 3 | `SP_QUERY_STUDENTS`, `SP_QUERY_STUDENT_SCORES`, `SP_DETECT_ABNORMAL` |
| 触发器 | 3 | `TRG_POLICY_CHANGE_LOG_INS/UPD/DEL` |
| 视图 | 1 | `V_MASKING_CONFIG` |

### 2.1 系统用户表 (`sys_user`)

```sql
DROP TABLE IF EXISTS sys_user_role;
DROP TABLE IF EXISTS sys_role_permission;

DROP TABLE IF EXISTS sensitive_field;

DROP TABLE IF EXISTS sys_role;
DROP TABLE IF EXISTS sys_user;

-- ============================================================
-- 1. sys_user  系统用户表
-- ============================================================
CREATE TABLE sys_user (
    id          BIGINT          PRIMARY KEY AUTO_INCREMENT  COMMENT '用户ID',
    username    VARCHAR(50)     NOT NULL UNIQUE             COMMENT '登录用户名',
    password    VARCHAR(255)    NOT NULL                    COMMENT '密码，BCrypt加密存储',
    real_name   VARCHAR(50)                                 COMMENT '真实姓名',
    status      TINYINT         DEFAULT 1                   COMMENT '状态：1正常，0禁用',
    create_time DATETIME        DEFAULT CURRENT_TIMESTAMP   COMMENT '创建时间',
    update_time DATETIME        DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) COMMENT='系统用户表';
```

### 2.4 用户角色关联表 (`sys_user_role`)

```sql

    user_id BIGINT  NOT NULL                    COMMENT '用户ID',
```


### 2.6 学生信息表 (`student_info`)

```sql
-- ============================================================
-- 6. student_info  学生信息表（业务核心数据源）
-- ============================================================
CREATE TABLE student_info (
    id              BIGINT          PRIMARY KEY AUTO_INCREMENT  COMMENT '主键ID',

    student_no      VARCHAR(30)     NOT NULL UNIQUE             COMMENT '学号',
    name            VARCHAR(50)                                 COMMENT '姓名',
    gender          CHAR(1)                                     COMMENT '性别：M-男，F-女，U-未知',
    phone           VARCHAR(20)                                 COMMENT '手机号',
    email           VARCHAR(100)                                COMMENT '邮箱',
    id_card         VARCHAR(30)                                 COMMENT '身份证号',
    address         VARCHAR(255)                                COMMENT '家庭住址',

    birth_date      DATE                                        COMMENT '出生日期，年龄由查询时计算',
    college         VARCHAR(100)                                COMMENT '学院',
    major           VARCHAR(100)                                COMMENT '专业',
    grade           VARCHAR(20)                                 COMMENT '年级',
    class_name      VARCHAR(100)                                COMMENT '班级',

    gpa             DECIMAL(3,2)                                COMMENT '绩点',
    family_income   DECIMAL(10,2)                               COMMENT '家庭年收入',
    bank_card       VARCHAR(30)                                 COMMENT '银行卡号',

    status          TINYINT         DEFAULT 1                   COMMENT '状态：1正常，0停用',
    create_time     DATETIME        DEFAULT CURRENT_TIMESTAMP   COMMENT '创建时间',
    update_time     DATETIME        DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    INDEX idx_college_major_grade (college, major, grade)       COMMENT '按学院/专业/年级条件查询',
    INDEX idx_student_no (student_no)                           COMMENT '按学号精确查询'
) COMMENT='学生信息表——系统脱敏保护的核心业务数据源。年龄不单独存储，通过 birth_date 计算得到，避免数据过期。';
```

### 2.8 敏感字段表 (`sensitive_field`)

```sql
-- ============================================================
-- 8. sensitive_field  敏感字段元数据表
-- ============================================================
CREATE TABLE sensitive_field (
    id              BIGINT          PRIMARY KEY AUTO_INCREMENT  COMMENT '敏感字段ID',
    entity_name     VARCHAR(50)     NOT NULL                    COMMENT '业务实体标签，如：STUDENT',
    table_name      VARCHAR(100)    NOT NULL                    COMMENT '数据库表名',
    column_name     VARCHAR(100)    NOT NULL                    COMMENT '数据库字段名',
    column_comment  VARCHAR(255)                                COMMENT '字段中文说明',
    sensitive_type  VARCHAR(50)     NOT NULL                    COMMENT '敏感类型，如：PHONE / ID_CARD / EMAIL / NAME / ADDRESS / INCOME / BANK_CARD / BIRTH_DATE',

    identify_method VARCHAR(50)                                 COMMENT '识别方式：MANUAL(手动) / FIELD_NAME(字段名匹配) / REGEX(正则表达式)',
    enabled         TINYINT         DEFAULT 1                   COMMENT '是否启用保护：1启用，0禁用',
    create_time     DATETIME        DEFAULT CURRENT_TIMESTAMP   COMMENT '创建时间',
    update_time     DATETIME        DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    UNIQUE KEY uk_field (table_name, column_name)               COMMENT '同一表的同一字段不可重复标记',

    INDEX idx_table_name (table_name)                           COMMENT '按表名查询敏感字段',
    INDEX idx_enabled (enabled)                                 COMMENT '按启用状态筛选'
) COMMENT='敏感字段元数据表——标记系统中哪些字段属于敏感数据及其等级。entity_name 提供业务语义层抽象。';
```

### 2.10 脱敏策略表 (`masking_policy`) ★新表

```sql
-- ============================================================
-- 10. masking_policy  脱敏策略表 ★（字段级：定义"怎么遮"）
-- ============================================================
CREATE TABLE masking_policy (
    id                  BIGINT          PRIMARY KEY AUTO_INCREMENT  COMMENT '策略ID',
    sensitive_field_id  BIGINT          NOT NULL                    COMMENT '敏感字段ID，确定"对哪个字段脱敏"',
    policy_name         VARCHAR(80)     NOT NULL                    COMMENT '策略名称，如：手机号保留前后缀_教师级',
    masking_type        VARCHAR(50)     NOT NULL                    COMMENT '脱敏方式编码',
    params              JSON                                        COMMENT '脱敏参数，须符合对应 masking_type 在 masking_type_dict 中定义的 param_schema',
    is_default          TINYINT         DEFAULT 0                   COMMENT '是否为字段级默认策略：1是，0否。每个字段至多一条默认策略，由应用层保证',
    status              TINYINT         DEFAULT 1                   COMMENT '状态：1启用，0禁用',
    create_by           BIGINT                                      COMMENT '创建人用户ID',
    update_by           BIGINT                                      COMMENT '更新人用户ID',
    create_time         DATETIME        DEFAULT CURRENT_TIMESTAMP   COMMENT '创建时间',
    update_time         DATETIME        DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    UNIQUE KEY uk_policy_field_name (sensitive_field_id, policy_name)   COMMENT '同一字段下的策略名不可重复',
    INDEX idx_sensitive_field_id (sensitive_field_id)                   COMMENT '按字段查策略',
    INDEX idx_masking_type (masking_type)                               COMMENT '按脱敏方式查策略',
    INDEX idx_is_default (is_default)                                   COMMENT '按默认策略筛选',

    CONSTRAINT fk_mp_sf  FOREIGN KEY (sensitive_field_id) REFERENCES sensitive_field(id)    ON DELETE RESTRICT,
    CONSTRAINT fk_mp_mt  FOREIGN KEY (masking_type)        REFERENCES masking_type_dict(type_code) ON DELETE RESTRICT,
    CONSTRAINT fk_mp_cb  FOREIGN KEY (create_by)           REFERENCES sys_user(id)          ON DELETE SET NULL,
    CONSTRAINT fk_mp_ub  FOREIGN KEY (update_by)           REFERENCES sys_user(id)          ON DELETE SET NULL
) COMMENT='脱敏策略表——定义字段级别的脱敏方式与参数。脱敏执行时：优先使用角色分配策略，无分配则回退到 is_default=1 的默认策略。';
```

### 2.11 脱敏规则分配表 (`masking_rule_assignment`) ★新表

```sql
-- ============================================================
-- 11. masking_rule_assignment  脱敏规则分配表 ★（角色级：定义"谁用哪条策略"）
-- ============================================================
CREATE TABLE masking_rule_assignment (
    id                  BIGINT          PRIMARY KEY AUTO_INCREMENT  COMMENT '分配ID',
    policy_id           BIGINT          NOT NULL                    COMMENT '策略ID，指向具体的脱敏策略',
    role_code           VARCHAR(50)     NOT NULL                    COMMENT '角色编码，如 TEACHER',
    sensitive_field_id  BIGINT          NOT NULL                    COMMENT '敏感字段ID（有意冗余，用于添加唯一约束见下方设计说明）',
    enabled             TINYINT         DEFAULT 1                   COMMENT '是否启用：1启用，0禁用',
    create_by           BIGINT                                      COMMENT '创建人用户ID',
    update_by           BIGINT                                      COMMENT '更新人用户ID',
    create_time         DATETIME        DEFAULT CURRENT_TIMESTAMP   COMMENT '创建时间',
    update_time         DATETIME        DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    UNIQUE KEY uk_assignment_field_role (sensitive_field_id, role_code) COMMENT '同一角色对同一字段至多分配一条策略',
    INDEX idx_policy_id (policy_id)                                     COMMENT '按策略查分配关系',
    INDEX idx_role_code (role_code)                                     COMMENT '按角色查所有分配',

    CONSTRAINT fk_mra_policy  FOREIGN KEY (policy_id)          REFERENCES masking_policy(id)   ON DELETE CASCADE,
    CONSTRAINT fk_mra_role    FOREIGN KEY (role_code)          REFERENCES sys_role(role_code)   ON DELETE RESTRICT,
    CONSTRAINT fk_mra_sf      FOREIGN KEY (sensitive_field_id) REFERENCES sensitive_field(id)   ON DELETE RESTRICT,
    CONSTRAINT fk_mra_cb      FOREIGN KEY (create_by)          REFERENCES sys_user(id)          ON DELETE SET NULL,
    CONSTRAINT fk_mra_ub      FOREIGN KEY (update_by)          REFERENCES sys_user(id)          ON DELETE SET NULL
) COMMENT='脱敏规则分配表——将脱敏策略分配给具体角色。

【设计说明】
sensitive_field_id 在此表中有意冗余。正常使用中可通过 policy_id → masking_policy → sensitive_field_id 获取该字段，
但为了添加 UNIQUE KEY (sensitive_field_id, role_code) 约束（保证同一角色对同一字段只有一条策略分配），
必须将该字段放在本表中。这是"以可控冗余换取数据完整性约束"的典型设计取舍。';
```

### 2.12 访问审计日志表 (`access_log`)

```sql
-- ============================================================
-- 12. access_log  访问审计日志表
-- ============================================================
CREATE TABLE access_log (
    id                BIGINT          PRIMARY KEY AUTO_INCREMENT  COMMENT '日志ID',
    user_id           BIGINT                                      COMMENT '用户ID',
    username          VARCHAR(50)                                 COMMENT '用户名（操作时快照，防止用户被删后日志丢失上下文）',
    role_code         VARCHAR(50)                                 COMMENT '角色编码（操作时快照）',
    operation_type    VARCHAR(50)                                 COMMENT '操作类型：QUERY(查询) / EXPORT(导出) / LOGIN(登录)',
    sql_text          TEXT                                        COMMENT '查询语句或查询条件（若为条件查询则记录条件JSON）',
    table_name        VARCHAR(100)                                COMMENT '访问的数据库表名',
    accessed_columns  TEXT                                        COMMENT '访问的字段列表，逗号分隔',
    sensitive_columns TEXT                                        COMMENT '涉及到的敏感字段列表，逗号分隔',
    masking_applied   TINYINT         DEFAULT 0                   COMMENT '是否执行了脱敏：1是，0否（如 SUPER_ADMIN 查看原始数据则为0）',
    masking_snapshot  JSON                                        COMMENT '脱敏结果快照：[{"field":"phone","policy_id":1,"masking_type":"KEEP_PREFIX_SUFFIX","before":"13812345678","after":"138****5678"}]',
    result_count      INT                                         COMMENT '返回记录数',
    client_ip         VARCHAR(100)                                COMMENT '客户端IP地址',
    access_time       DATETIME        DEFAULT CURRENT_TIMESTAMP   COMMENT '访问时间',

    INDEX idx_user_time (user_id, access_time)                    COMMENT '按用户+时间查询审计日志（高频场景）',
    INDEX idx_role_time (role_code, access_time)                  COMMENT '按角色+时间查询审计日志',
    INDEX idx_table_name (table_name)                             COMMENT '按表名查访问记录',
    INDEX idx_operation_type (operation_type)                     COMMENT '按操作类型筛选',

    CONSTRAINT fk_al_user FOREIGN KEY (user_id) REFERENCES sys_user(id) ON DELETE SET NULL
) COMMENT='访问审计日志表——记录每一次敏感数据访问的完整信息。

【字段说明】
- masking_snapshot：存储 JSON 数组，每条记录描述一个字段的脱敏前后值、使用的策略和脱敏方式。
  用于审计时回溯"当时用户看到了什么"而非仅记录"应用了哪些规则"。
- username / role_code：快照字段。指向的用户可能被删除，但日志需要保留操作时的身份信息。';
```

### 2.13 策略变更日志表 (`rule_change_log`)

```sql
-- ============================================================
-- 13. rule_change_log  脱敏策略变更日志表
-- ============================================================
CREATE TABLE rule_change_log (
    id              BIGINT          PRIMARY KEY AUTO_INCREMENT  COMMENT '日志ID',
    policy_id       BIGINT                                      COMMENT '变更的策略ID',
    operator_id     BIGINT                                      COMMENT '操作人用户ID',
    operator_name   VARCHAR(50)                                 COMMENT '操作人名称（操作时快照）',
    operation_type  VARCHAR(50)                                 COMMENT '操作类型：CREATE(新建) / UPDATE(修改) / DELETE(删除) / ENABLE(启用) / DISABLE(禁用)',
```

### 5.1 脱敏函数 `FN_APPLY_MASK`

```sql
-- ============================================================
-- FN_APPLY_MASK  统一脱敏函数
-- 功能：根据脱敏方式和参数对输入值执行脱敏转换
-- 参数：
--   p_value      VARCHAR   待脱敏的原始值（NULL 时返回 NULL）
--   p_mask_type  VARCHAR   脱敏方式编码（来自 masking_type_dict.type_code）
--   p_params     JSON      脱敏参数
-- 返回：脱敏后的字符串
-- ============================================================

DROP FUNCTION IF EXISTS FN_APPLY_MASK;

DELIMITER $$

CREATE FUNCTION FN_APPLY_MASK(
    p_value     VARCHAR(500),
    p_mask_type VARCHAR(50),
    p_params    JSON
)
RETURNS VARCHAR(500)
DETERMINISTIC
READS SQL DATA
COMMENT '统一脱敏函数：根据 type + params 对输入值执行对应脱敏算法'
BEGIN
    DECLARE v_result   VARCHAR(500);
    DECLARE v_len      INT;
    DECLARE v_prefix   INT;
    DECLARE v_suffix   INT;
    DECLARE v_step     INT;
    DECLARE v_level    VARCHAR(20);
    DECLARE v_at_pos   INT;

    -- 空值直接返回
    IF p_value IS NULL THEN
        RETURN NULL;
    END IF;

    SET v_len = CHAR_LENGTH(p_value);

    CASE p_mask_type

        -- 不脱敏
        WHEN 'NO_MASK' THEN
            RETURN p_value;

        -- 全遮蔽
        WHEN 'FULL_MASK' THEN
            RETURN REPEAT('*', v_len);

        -- 保留前 N 位
        WHEN 'KEEP_PREFIX' THEN
            SET v_prefix = COALESCE(JSON_UNQUOTE(JSON_EXTRACT(p_params, '$.prefix')), 1);
            IF v_prefix >= v_len THEN
                RETURN p_value;
            END IF;
            RETURN CONCAT(LEFT(p_value, v_prefix), REPEAT('*', v_len - v_prefix));

        -- 保留后 N 位
        WHEN 'KEEP_SUFFIX' THEN
            SET v_suffix = COALESCE(JSON_UNQUOTE(JSON_EXTRACT(p_params, '$.suffix')), 1);
            IF v_suffix >= v_len THEN
                RETURN p_value;
            END IF;
            RETURN CONCAT(REPEAT('*', v_len - v_suffix), RIGHT(p_value, v_suffix));

        -- 保留前后位
        WHEN 'KEEP_PREFIX_SUFFIX' THEN
            SET v_prefix = COALESCE(JSON_UNQUOTE(JSON_EXTRACT(p_params, '$.prefix')), 3);
            SET v_suffix = COALESCE(JSON_UNQUOTE(JSON_EXTRACT(p_params, '$.suffix')), 4);
            IF v_prefix + v_suffix >= v_len THEN
                RETURN p_value;
            END IF;
            RETURN CONCAT(
                LEFT(p_value, v_prefix),
                REPEAT('*', v_len - v_prefix - v_suffix),
                RIGHT(p_value, v_suffix)
            );

        -- 邮箱脱敏：用户名部分保留首字符，其余遮蔽；域名保留
        WHEN 'EMAIL_MASK' THEN
            SET v_at_pos = INSTR(p_value, '@');
            IF v_at_pos <= 1 THEN
                RETURN REPEAT('*', v_len);
            END IF;
            RETURN CONCAT(
                LEFT(p_value, 1),
                REPEAT('*', v_at_pos - 2),
                SUBSTRING(p_value, v_at_pos)
            );

        -- 地址层级脱敏：保留省/市，其余遮蔽
        WHEN 'ADDRESS_LEVEL' THEN
            SET v_level = COALESCE(JSON_UNQUOTE(JSON_EXTRACT(p_params, '$.level')), 'city');
            IF v_level = 'province' THEN
                RETURN CONCAT(LEFT(p_value, 3), REPEAT('*', IF(v_len > 3, v_len - 3, 0)));
            ELSEIF v_level = 'city' THEN
                RETURN CONCAT(LEFT(p_value, 6), REPEAT('*', IF(v_len > 6, v_len - 6, 0)));
            ELSE
                RETURN CONCAT(LEFT(p_value, 3), REPEAT('*', IF(v_len > 3, v_len - 3, 0)));
            END IF;

        -- 区间泛化：将数值向下取整到指定步长
        WHEN 'GENERALIZATION' THEN
            SET v_step = COALESCE(JSON_UNQUOTE(JSON_EXTRACT(p_params, '$.step')), 10000);
            RETURN CONCAT(
                FLOOR(CAST(p_value AS DECIMAL(12,2)) / v_step) * v_step,
                '-',
                (FLOOR(CAST(p_value AS DECIMAL(12,2)) / v_step) + 1) * v_step
            );

        -- 哈希脱敏：返回 MD5 前 8 位
        WHEN 'HASH_MASK' THEN
            RETURN LEFT(MD5(p_value), 8);

        -- 保留年份：日期类型仅保留年份
        WHEN 'KEEP_YEAR' THEN
            RETURN CONCAT(LEFT(p_value, 4), '-**-**');

        -- 数值加噪：原值 ± 指定百分比范围
        WHEN 'NOISE' THEN
            SET v_step = COALESCE(JSON_UNQUOTE(JSON_EXTRACT(p_params, '$.percent')), 10);
            RETURN CONCAT(
                ROUND(CAST(p_value AS DECIMAL(12,2)) * (1 - v_step / 100), 2),
                '-',
                ROUND(CAST(p_value AS DECIMAL(12,2)) * (1 + v_step / 100), 2)
            );

        -- 未知脱敏方式：返回全遮蔽作为安全兜底
        ELSE
            RETURN REPEAT('*', v_len);

    END CASE;

END$$

DELIMITER ;
```

### 5.2 查询存储过程 `SP_QUERY_STUDENTS`

```sql
-- ============================================================
-- SP_QUERY_STUDENTS  学生信息动态脱敏查询存储过程
-- 功能：根据角色编码查询学生信息，自动应用脱敏策略
-- 参数：
--   p_role_code  VARCHAR(50)   角色编码
-- ============================================================

DROP PROCEDURE IF EXISTS SP_QUERY_STUDENTS;

DELIMITER $$

CREATE PROCEDURE SP_QUERY_STUDENTS(
    IN p_role_code VARCHAR(50)
)
COMMENT '根据角色编码查询 student_info，自动应用脱敏策略。优先使用角色分配策略，无分配则回退到字段默认策略。'
BEGIN
    -- 声明游标变量，用于读取每个敏感字段的脱敏配置
    DECLARE v_done        INT DEFAULT 0;
    DECLARE v_col         VARCHAR(100);
    DECLARE v_type        VARCHAR(50);
    DECLARE v_params_str  VARCHAR(500);
    DECLARE v_sql_parts   TEXT DEFAULT '';
    DECLARE v_sql         TEXT;
    DECLARE v_match_count INT DEFAULT 0;

    -- 游标：查询当前角色可用的脱敏配置
    -- 优先取角色分配策略（通过 masking_rule_assignment），无则取字段默认策略（is_default=1）
    DECLARE cur CURSOR FOR
        SELECT
            sf.column_name,
            COALESCE(assign.masking_type, def.masking_type)  AS masking_type,
            COALESCE(assign.params, def.params)              AS params
        FROM sensitive_field sf
        -- 字段默认策略
        LEFT JOIN masking_policy def
            ON def.sensitive_field_id = sf.id
            AND def.is_default = 1
            AND def.status = 1
        -- 角色分配策略（有则覆盖默认）
        LEFT JOIN (
            SELECT sf2.id AS field_id, mp.masking_type, mp.params
            FROM masking_rule_assignment mra
            JOIN masking_policy mp ON mp.id = mra.policy_id AND mp.status = 1
            JOIN sensitive_field sf2 ON sf2.id = mra.sensitive_field_id
            WHERE mra.role_code = p_role_code
              AND mra.enabled = 1
        ) assign ON assign.field_id = sf.id
        WHERE sf.table_name = 'student_info'
          AND sf.enabled = 1
        ORDER BY sf.id;

    DECLARE CONTINUE HANDLER FOR NOT FOUND SET v_done = 1;

    -- 遍历每个敏感字段，构造 SELECT 片段
    OPEN cur;
    read_loop: LOOP
        FETCH cur INTO v_col, v_type, v_params_str;
        IF v_done THEN
            LEAVE read_loop;
        END IF;

        -- 有脱敏配置时，用 FN_APPLY_MASK 包裹；无配置则直接暴露原始值
        IF v_type IS NOT NULL THEN
            SET v_sql_parts = CONCAT(v_sql_parts,
                '    FN_APPLY_MASK(si.', v_col,
                ', ''', v_type, ''', ',
                IFNULL(CONCAT('''', v_params_str, ''''), '''{}'''),
                ') AS ', v_col, ',\n');
            SET v_match_count = v_match_count + 1;
        ELSE
            SET v_sql_parts = CONCAT(v_sql_parts, '    si.', v_col, ',\n');
        END IF;
    END LOOP;
    CLOSE cur;

    -- 拼接完整 SQL
    -- 包含所有字段：脱敏字段 + 非敏感字段
    SET v_sql = CONCAT(
        'SELECT si.id,\n',
        '       si.student_no,\n',
        '       si.gender,\n',
        v_sql_parts,
        '       si.college,\n',
        '       si.major,\n',
        '       si.grade,\n',
        '       si.class_name,\n',
        '       TIMESTAMPDIFF(YEAR, si.birth_date, CURDATE()) AS age,\n',
        '       si.gpa,\n',
        '       si.status,\n',
        '       si.create_time,\n',
        '       si.update_time\n',
        'FROM student_info si\n',
        'WHERE si.status = 1\n',
        'ORDER BY si.id'
    );

    -- 使用 PREPARE/EXECUTE 执行动态 SQL
    SET @sql = v_sql;
    PREPARE stmt FROM @sql;
    EXECUTE stmt;
    DEALLOCATE PREPARE stmt;

    -- 写入审计日志
    INSERT INTO access_log (
        role_code, operation_type, table_name,
        masking_applied, result_count, access_time
    ) VALUES (
        p_role_code, 'QUERY', 'student_info',
        IF(v_match_count > 0, 1, 0),
        (SELECT COUNT(*) FROM student_info WHERE status = 1),
        NOW()
    );

END$$

DELIMITER ;
```

### 5.2b 成绩查询存储过程 `SP_QUERY_STUDENT_SCORES`

```sql
-- SP_QUERY_STUDENT_SCORES  学生成绩动态脱敏查询存储过程
-- 功能：根据角色编码查询学生成绩，自动对 score 字段应用脱敏
-- 参数：
--   p_role_code  VARCHAR(50)   角色编码
-- ============================================================

DROP PROCEDURE IF EXISTS SP_QUERY_STUDENT_SCORES;

DELIMITER $$

CREATE PROCEDURE SP_QUERY_STUDENT_SCORES(
    IN p_role_code VARCHAR(50)
)
COMMENT '根据角色编码查询 student_score，自动对成绩字段应用脱敏策略'
BEGIN
    DECLARE v_mask_type VARCHAR(50);
    DECLARE v_params    VARCHAR(500);
    DECLARE v_sql       TEXT;

    -- 查找成绩字段的脱敏策略（优先角色分配，无则默认策略）
    SELECT COALESCE(assign.masking_type, def.masking_type),
           COALESCE(assign.params, def.params)
    INTO v_mask_type, v_params
    FROM sensitive_field sf
    LEFT JOIN masking_policy def
        ON def.sensitive_field_id = sf.id AND def.is_default = 1 AND def.status = 1
    LEFT JOIN (
        SELECT sf2.id AS field_id, mp.masking_type, mp.params
        FROM masking_rule_assignment mra
        JOIN masking_policy mp ON mp.id = mra.policy_id AND mp.status = 1
        JOIN sensitive_field sf2 ON sf2.id = mra.sensitive_field_id
        WHERE mra.role_code = p_role_code AND mra.enabled = 1
    ) assign ON assign.field_id = sf.id
    WHERE sf.table_name = 'student_score'
      AND sf.column_name = 'score'
      AND sf.enabled = 1
    LIMIT 1;

    -- 动态构造查询：根据是否有脱敏策略决定是否调用 FN_APPLY_MASK
    IF v_mask_type IS NOT NULL THEN
        SET @sql = CONCAT(
            'SELECT ss.id, ss.student_id, si.student_no, si.name AS student_name, ',
            'ss.course_name, ',
            'FN_APPLY_MASK(ss.score, ''', v_mask_type, ''', ''',
            IFNULL(v_params, '{}'), ''') AS score, ',
            'ss.semester, ss.create_time ',
            'FROM student_score ss ',
            'JOIN student_info si ON si.id = ss.student_id ',
            'WHERE si.status = 1 ',
            'ORDER BY ss.student_id, ss.course_name'
        );
    ELSE
        SET @sql = CONCAT(
            'SELECT ss.id, ss.student_id, si.student_no, si.name AS student_name, ',
            'ss.course_name, ss.score, ss.semester, ss.create_time ',
            'FROM student_score ss ',
            'JOIN student_info si ON si.id = ss.student_id ',
            'WHERE si.status = 1 ',
            'ORDER BY ss.student_id, ss.course_name'
        );
    END IF;

    PREPARE stmt FROM @sql;
    EXECUTE stmt;
    DEALLOCATE PREPARE stmt;

    -- 写入审计日志
    INSERT INTO access_log (role_code, operation_type, table_name, masking_applied, result_count, access_time)
    VALUES (
        p_role_code, 'QUERY', 'student_score',
        IF(v_mask_type IS NOT NULL, 1, 0),
        (SELECT COUNT(*) FROM student_score),
        NOW()
    );

END$$

DELIMITER ;
```

### 5.3 异常检测存储过程 `SP_DETECT_ABNORMAL`

```sql
-- ============================================================
-- SP_DETECT_ABNORMAL  异常访问检测存储过程
-- 功能：定时执行，扫描 access_log 检测异常行为并写入 abnormal_access
-- 调用方式：CREATE EVENT / MySQL 定时任务 / Java @Scheduled 调用
-- ============================================================

DROP PROCEDURE IF EXISTS SP_DETECT_ABNORMAL;

DELIMITER $$

CREATE PROCEDURE SP_DETECT_ABNORMAL()
COMMENT '异常访问检测：扫描 access_log 发现高频查询、敏感字段集中访问等异常行为'
BEGIN

    -- 1. 检测高频查询：同一用户 1 小时内查询超过 100 次
    INSERT INTO abnormal_access (user_id, username, role_code, abnormal_type, trigger_source, detail, severity)
    SELECT
        al.user_id,
        al.username,
        al.role_code,
        'HIGH_FREQ',
        'API',
        CONCAT('1小时内查询了 ', COUNT(*), ' 次 student_info 表'),
        CASE WHEN COUNT(*) > 200 THEN 'HIGH' ELSE 'MEDIUM' END
    FROM access_log al
    WHERE al.operation_type = 'QUERY'
      AND al.access_time >= DATE_SUB(NOW(), INTERVAL 1 HOUR)
      AND al.user_id IS NOT NULL
    GROUP BY al.user_id, al.username, al.role_code
    HAVING COUNT(*) > 100;

    -- 2. 检测敏感字段集中访问：1 天内查询 id_card 或 bank_card 超过 50 次
    INSERT INTO abnormal_access (user_id, username, role_code, abnormal_type, trigger_source, detail, severity)
    SELECT
        al.user_id,
        al.username,
        al.role_code,
        'SENSITIVE_FOCUS',
        'API',
        CONCAT('1天内 ', COUNT(*), ' 次查询涉及高敏感字段（id_card / bank_card）'),
        'HIGH'
    FROM access_log al
    WHERE al.operation_type = 'QUERY'
      AND al.access_time >= DATE_SUB(NOW(), INTERVAL 1 DAY)
      AND (
          al.sensitive_columns LIKE '%id_card%'
          OR al.sensitive_columns LIKE '%bank_card%'
      )
      AND al.user_id IS NOT NULL
    GROUP BY al.user_id, al.username, al.role_code
    HAVING COUNT(*) > 50;

    -- 3. 检测大量数据导出：单次操作导出超过 500 条记录
    INSERT INTO abnormal_access (user_id, username, role_code, abnormal_type, trigger_source, detail, severity)
    SELECT
        al.user_id,
        al.username,
        al.role_code,
        'LARGE_EXPORT',
        'EXPORT',
        CONCAT('单次导出 ', al.result_count, ' 条记录'),
        CASE WHEN al.result_count > 1000 THEN 'HIGH' ELSE 'MEDIUM' END
    FROM access_log al
    WHERE al.operation_type = 'EXPORT'
      AND al.result_count > 500
      AND al.access_time >= DATE_SUB(NOW(), INTERVAL 1 DAY)
      AND al.user_id IS NOT NULL;

    -- 4. 检测普通用户越权尝试：NORMAL 角色尝试访问高敏感字段
    INSERT INTO abnormal_access (user_id, username, role_code, abnormal_type, trigger_source, detail, severity)
    SELECT
        al.user_id,
        al.username,
        al.role_code,
        'OVER_AUTH',
        'API',
        CONCAT('NORMAL 用户访问高敏感字段：', al.sensitive_columns),
        'HIGH'
    FROM access_log al
    WHERE al.role_code = 'NORMAL'
      AND al.sensitive_columns IS NOT NULL
      AND al.sensitive_columns != ''
      AND al.access_time >= DATE_SUB(NOW(), INTERVAL 1 DAY)
      AND al.user_id IS NOT NULL;

END$$

DELIMITER ;
```

```sql
-- ============================================================
-- TRG_POLICY_CHANGE_LOG_INS / UPD / DEL
-- 功能：masking_policy 表发生增/改/删时，自动写入 rule_change_log
-- 说明：依赖会话变量 @current_user_id 获取操作人ID
--       Java 后端在执行策略变更前需 SET @current_user_id = ?
-- ============================================================

DROP TRIGGER IF EXISTS TRG_POLICY_CHANGE_LOG_INS;
DROP TRIGGER IF EXISTS TRG_POLICY_CHANGE_LOG_UPD;
DROP TRIGGER IF EXISTS TRG_POLICY_CHANGE_LOG_DEL;

DELIMITER $$

-- INSERT 触发器
CREATE TRIGGER TRG_POLICY_CHANGE_LOG_INS
AFTER INSERT ON masking_policy
FOR EACH ROW
BEGIN
    INSERT INTO rule_change_log (
        policy_id, operator_id, operator_name,
        operation_type, after_content, remark
    ) VALUES (
        NEW.id,
        @current_user_id,
        (SELECT real_name FROM sys_user WHERE id = @current_user_id),
        'CREATE',
        JSON_OBJECT(
            'policy_name',     NEW.policy_name,
            'sensitive_field_id', NEW.sensitive_field_id,
            'masking_type',    NEW.masking_type,
            'params',          NEW.params,
            'is_default',      NEW.is_default,
            'status',          NEW.status
        ),
        '新建脱敏策略'
    );
END$$

-- UPDATE 触发器
CREATE TRIGGER TRG_POLICY_CHANGE_LOG_UPD
AFTER UPDATE ON masking_policy
FOR EACH ROW
BEGIN
    INSERT INTO rule_change_log (
        policy_id, operator_id, operator_name,
        operation_type, before_content, after_content, remark
    ) VALUES (
        NEW.id,
        @current_user_id,
        (SELECT real_name FROM sys_user WHERE id = @current_user_id),
        IF(NEW.status != OLD.status,
           IF(NEW.status = 1, 'ENABLE', 'DISABLE'),
           'UPDATE'
        ),
        JSON_OBJECT(
            'policy_name',     OLD.policy_name,
            'sensitive_field_id', OLD.sensitive_field_id,
            'masking_type',    OLD.masking_type,
            'params',          OLD.params,
            'is_default',      OLD.is_default,

        JSON_OBJECT(
            'policy_name',     NEW.policy_name,
            'sensitive_field_id', NEW.sensitive_field_id,
            'masking_type',    NEW.masking_type,
            'params',          NEW.params,
            'is_default',      NEW.is_default,
            'status',          NEW.status
        ),
        '修改脱敏策略'
    );
END$$

-- DELETE 触发器

        @current_user_id,
        (SELECT real_name FROM sys_user WHERE id = @current_user_id),

            'policy_name',     OLD.policy_name,
            'sensitive_field_id', OLD.sensitive_field_id,
            'masking_type',    OLD.masking_type,
            'params',          OLD.params,
            'is_default',      OLD.is_default,
            'status',          OLD.status
        ),
        '删除脱敏策略'
    );
END$$

DELIMITER ;
```

### 5.5 脱敏配置视图 `V_MASKING_CONFIG`

```sql
-- ============================================================
-- V_MASKING_CONFIG  脱敏配置总览视图
-- 功能：将敏感字段、脱敏策略、角色分配扁平化展示，便于管理页面查询和调试
-- ============================================================

DROP VIEW IF EXISTS V_MASKING_CONFIG;

CREATE VIEW V_MASKING_CONFIG AS
SELECT
    sf.id               AS sensitive_field_id,
    sf.entity_name,
    sf.table_name,
    sf.column_name,
    sf.column_comment,
    sf.sensitive_type,
    sf.sensitive_level,
    sf.enabled          AS field_enabled,

    mp.id               AS policy_id,
    mp.policy_name,
    mp.masking_type,
    mp.params,
    mp.is_default,
    mp.status           AS policy_status,

    mtd.type_name       AS masking_type_name,
    mtd.param_schema,
    mtd.default_params,

    mra.id              AS assignment_id,
    mra.role_code       AS assigned_role,
    mra.enabled         AS assignment_enabled

FROM sensitive_field sf
LEFT JOIN masking_policy mp
    ON mp.sensitive_field_id = sf.id AND mp.status = 1
LEFT JOIN masking_type_dict mtd
    ON mtd.type_code = mp.masking_type AND mtd.status = 1
LEFT JOIN masking_rule_assignment mra
    ON mra.policy_id = mp.id AND mra.enabled = 1
WHERE sf.enabled = 1
ORDER BY sf.id, mp.is_default DESC, mp.id;
```

## 七、架构说明

### 7.1 动态脱敏查询流程

```
用户请求 (role_code = 'TEACHER')
    │
    ▼
┌────────────────────────────────────────────┐
│ Java 后端（Spring Boot + Spring Security）   │
│  1. JWT 认证 → 解析 user_id                │
│  2. 查 sys_user_role → 获取 role_code      │
│  3. 查 sys_role_permission → 权限鉴权       │
│  4. 权限通过 → CALL SP_QUERY_STUDENTS()           │
│     （或 CALL SP_QUERY_STUDENT_SCORES() 查成绩）    │
└──────────────────┬─────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────────────────────┐
│ MySQL SP_QUERY_STUDENTS(p_role_code = 'TEACHER')             │
│                                                              │
│  1. 遍历 sensitive_field（WHERE table_name='student_info'）   │
│  2. 对每个敏感字段，查询脱敏策略：                             │
│     a. 先查 masking_rule_assignment（角色分配策略）           │
│     b. 无分配则 fallback masking_policy.is_default=1          │
│  3. 动态构造 SELECT 语句：                                    │
│     - 有策略的字段 → FN_APPLY_MASK(col, type, params)        │
│     - 无策略的字段 → 原始列                                   │
│  4. PREPARE → EXECUTE 执行动态 SQL                           │
│  5. INSERT INTO access_log（审计日志）                        │
│  6. 返回已脱敏结果集给 Java 后端                              │
└──────────────────────────────────────────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────┐
│ 独立机制（异步/定时）                          │
│                                              │
│  触发器 TRG_POLICY_CHANGE_LOG_*               │
│    → 任何对 masking_policy 的 CUD 操作        │
│    → 自动写入 rule_change_log                │
│                                              │
│  存储过程 SP_DETECT_ABNORMAL()               │
│    → 定时扫描 access_log                     │
│    → 检测异常模式 → 写入 abnormal_access       │
│    → 可通过 MySQL EVENT 或 Java @Scheduled 触发│
└──────────────────────────────────────────────┘
```

### 7.2 策略优先级

```
查询脱敏规则时的优先级（由高到低）：

1. masking_rule_assignment (role_code = 'TEACHER', enabled = 1)
     ↓ 无匹配
2. masking_policy (is_default = 1, status = 1)
     ↓ 无匹配
3. 返回原始值（不脱敏）
```

### 7.3 关键技术决策

| 决策 | 选项 | 理由 |
|---|---|---|
| 脱敏计算位置 | MySQL 函数 + 存储过程 | 数据库课程设计的核心考点，展示函数/SP/触发器/视图的综合运用 |
| 脱敏规则存储 | 策略表 + 分配表分离 | 语义清晰："字段怎么遮"(policy) vs "谁用哪条"(assignment)，便于扩展 |
| 审计日志写入 | 存储过程中直接 INSERT | 日志与查询在同一事务中，保证一致性 |
| 策略变更日志 | 触发器自动写入 | 减少 Java 代码侵入，保证变更记录 100% 覆盖 |
| 异常检测 | 独立存储过程，定时触发 | 解耦检测逻辑与查询逻辑，避免查询性能受影响 |
| 年龄字段 | 只存 birth_date，查询时计算 | 避免数据过期 |
| 成绩拆分 | student_score 独立表，FK 关联 student_info | 符合 3NF，支持成绩脱敏，避免 student_info 过胖 |
